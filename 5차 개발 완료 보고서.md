# 5차 개발 완료 보고서 (사전환경조사서 및 자산목록 데이터 자동 병합 기능 고도화)

본 보고서는 **정보보안감사 보안솔루션 로드맵 자동 수립 시스템**의 5차 개발 단계에서 수행된 사전환경조사서 및 자산목록/중요도 평가서 자료의 업로드 저장소 구축, 실시간 데이터 파싱, 그리고 최종 로드맵 엑셀 생성 시의 동적 데이터 병합(Merge) 및 포맷 무결성 복구 알고리즘 구현의 최종 결과를 기술합니다.

---

## 1. 개발 개요
* **목표**:
  - 사전환경조사서 엑셀 파일을 업로드 및 분석하여 기업 정보(기업명, 사업분야 등)를 최종 로드맵 엑셀 상단 영역에 자동 매핑.
  - 자산목록 및 중요도 평가서의 `4.정보보호시스템` 시트 내 실시간 가동 중인 보안 장비 목록을 읽어 로드맵 하단의 "보안 솔루션 세부 현황" 테이블에 자동 이식하고, 비고란에는 자산 중요도 등급 정보(`기밀성|무결성|가용성|합계|등급`)를 조합하여 바인딩.
  - 프론트엔드 UI 상에서 세 가지 진단 자료(상세체크리스트, 사전환경조사서, 자산목록 및 중요도 평가서)를 편리하게 관리하고 업로드할 수 있도록 Premium Grid 레이아웃 드롭존 리디자인.
  - 최종 엑셀 추출 시 가변적으로 변동하는 보안 자산 개수($N$개)에 맞춰 표의 행을 삽입/삭제하고 스타일 서식 및 셀 병합 영역을 완벽하게 재구성하는 동적 엑셀 정렬 알고리즘 구축.
  - 연속적인 다중 클릭으로 인한 중복 API 호출 방지 가드 전역 적용.
* **수행 기간**: 2026년 6월 1일
* **핵심 구현체**:
  - `app.py` (사전환경조사서 `/api/upload_env` 및 자산목록 `/api/upload_asset` 업로드 API 신설, `/api/export` 파일 경로 수신 파라미터 파이프라인 연동)
  - `roadmap_agent_llm_new.py` (사전환경조사서 파싱, 자산목록 파싱, `generate_roadmap_excel` 메소드 내 동적 행 삽입/삭제 및 스타일 이식, 행 높이 복구, 헤더 변경 알고리즘 구현)
  - `templates/index.html` (3개 파일 동시 접수를 위한 3-Column Responsive Grid 드롭존 리디자인 및 아이콘 최적화)
  - `static/js/app.js` (다중 비동기 파일 업로드 바인딩 및 파일 경로 전역 메모리 관리, 다운로드 폼 매칭, 중복 클릭 가드 확대 적용)
  - `static/css/style.css` (사전환경조사서/자산목록 드롭존 전용 커스텀 호버 및 드래그오버 스타일링 탑재)

## 2. 5차 개발 핵심 항목 및 구현 내용

### ① 사전환경조사서 자동 기업 정보 파싱 및 바인딩
* **요구 분석**: 기존에는 엑셀 템플릿의 기업 정보 영역이 기존 제로인 데이터로 고정되어 있었으나, 사용자가 제출한 사전환경조사서의 정보를 파싱하여 실시간으로 업체 정보가 최종 로드맵에 적용되어야 했습니다.
* **구현 내용**: 
  - 업로드된 파일의 2번 시트(이름에 `OO그룹`이 포함되어 있거나 index 2인 시트)의 신청업체 정보 병합 셀 구조를 완벽 분석했습니다.
  - `기업명`, `사업 형태`, `임직원 수`, `주요 적용 법률` (Col B, Row 5~8), `사업분야`, `정보통신 사업자 여부`, `전년도 매출`, `감독기관` (Col H, Row 5~8)의 값을 정확히 파싱합니다.
  - 천 단위 정수 콤마 포맷팅과 '명' 등의 단위 보정 헬퍼 함수(`clean_env_value`)를 적용하여 정돈된 형태의 문자열로 최종 로드맵의 `B5~B8` 및 `I5~I8` 병합 셀 영역에 자동으로 써넣도록 연동 완료했습니다.

### ② 보안 솔루션 자산 현황 추출 및 자산중요도 지표 셀 개별 분류 기입
* **요구 분석**: 
  - 자산목록에서 유효하지 않은 빈 데이터 행(구분, IP, 호스트명, 벤더가 모두 없는 행)이 병합 과정에서 포함되어 표시되는 오류를 차단해야 했습니다.
  - 기존에 단일 셀(비고란)에 파이프라인(`|`) 형태로 압축해 기입했던 기밀성/무결성/가용성/합계/등급 정보를 엑셀의 각 독립된 셀로 분류하여 매핑하고, 자동 합산 수식 및 합계에 따른 등급 지표 산정이 필요했습니다.
  - 자산 목록 중 특별히 배경색이 빨간색(Red Fill)으로 지정되어 실가동되지 않는 자산은 비고란에 특정 표시를 남겨 식별성을 확보해야 했습니다.
* **구현 내용**:
  - `4.정보보호시스템` 시트 파싱 시, `구분`, `IP 주소`, `호스트명`, `벤더 및 모델명`이 모두 비어있는 행은 자동 스킵(`continue`)되도록 처리했습니다.
  - 최종 엑셀의 Row 12, Column 9~14 영역의 기존 가로 병합을 지우고, 각각 `"기밀성"`, `"무결성"`, `"가용성"`, `"합계"`, `"등급"`, `"비고"`로 개별 컬럼 헤더 명칭을 부여하여 분류했습니다.
  - 각 자산의 데이터 행에 대해 기밀성(Col 9), 무결성(Col 10), 가용성(Col 11) 수치를 정수형으로 기입하고, 합계(Col 12) 셀에는 `=SUM(I{row}:K{row})` 형태의 수식을 삽입하여 자동 합산이 작동하도록 구현했습니다.
  - 기밀성, 무결성, 가용성 합산 점수를 기반으로 등급(Col 13) 지표를 자동 산정했습니다 (7~9점: `H`, 5~6점: `M`, 3~4점: `L`).
  - 자산목록 원본 시트 내 해당 자산 행의 특정 셀 배경색이 빨간색 계열로 지정되어 있는 자산을 감지하는 `is_red_fill` 판단 모듈을 구현하고, 빨간색 배경을 지닌 자산에 대해 최종 비고란(Col 14~16 병합 영역)에 `운영하고 있지 않는 자산`이라는 경고 문구를 자동으로 써넣도록 매핑하였습니다. 그 외 정상 자산은 비고란이 비어있도록 설정했습니다.

### ③ 동적 행 삽입/삭제, 배경색 이식 및 행 높이/열 너비/셀 스타일 정렬 알고리즘
* **요구 분석**: 
  - openpyxl 조작 중 행 높이 정보가 왜곡되어 자산목록 행이 비정상적으로 비대해지는 문제를 해결하고 모든 데이터 행의 너비/높이를 균일하게 정렬해야 했습니다.
  - 자산목록 원본 엑셀에 칠해진 특정 셀의 색상을 최종 로드맵 엑셀에도 완벽하게 시각적으로 보존해야 했습니다.
  - **[보완]** 새로 추가된 기밀성~비고(Col 9~16) 셀 영역에 헤더 스타일이 유실되고 테두리가 깨지는 문제를 해결하여 디자인 일관성을 확보하고, 셀 내부의 값을 단락 정렬 대신 "가운데 정렬(중간 맞춤)"로 깔끔하게 통일해야 했습니다.
* **구현 내용**:
  - 행 조작 시작 전, 원본 템플릿의 모든 행 높이(`row_dimensions[r].height`) 딕셔너리를 백업하는 알고리즘을 탑재했습니다.
  - 행의 추가/삭제가 완료된 후, 자산 목록 데이터가 들어가는 모든 행(Row 13 ~ Row 13 + N - 1)의 높이를 템플릿 13행의 표준 높이(23.25)로 고정하여 비대해지는 레이아웃 문제를 완벽하게 통일 정렬시켰습니다.
  - 엑셀 파일 내의 자산 데이터 열(No, 구분, IP주소, 벤더 및 모델명, 기능, 자산위치, 담당자, 담당부서)의 너비(Width)를 각각 내용에 맞추어 표준화된 크기(6~30)로 재지정하여, 글자가 잘리거나 여백이 과해지는 현상을 해결했습니다.
  - 밀려난 23행 이상의 본문 행들에 대해서는 오프셋 `diff`를 계산·가산하여 백업해두었던 행 높이들을 올바른 인덱스에 복구시켰습니다.
  - **[보완]** 12행 헤더 셀(Col 9~16)에 Col 2(구분)의 채우기 색상, 글꼴, 테두리를 고스란히 복제 이식하고 정렬은 가운데 정렬로 지정하여 일관된 12행 헤더 디자인을 구현했습니다.
  - **[보완]** 데이터 행들의 신설된 Col 9~16 셀 전체에 Col 8(담당부서)의 테두리(Border) 속성을 복사 적용하고, 가로/세로 가운데 정렬(Center Alignment)을 강제 적용해 가독성을 개선했습니다.
  - **[보완]** 행 배경색의 일관성을 위해 원본 자산목록 구분 셀에 배경색이 칠해져 있을 경우, 신설된 기밀성~비고(Col 9~16) 데이터 영역까지 동일한 색상을 자동 복제 확장하여 전체 행의 색상 통일성을 이끌어 냈습니다.
  - 자산목록 파싱 시 각 셀의 `fill`(PatternFill) 속성을 함께 추출하고, 최종 엑셀 쓰기 시 배경색이 실제로 지정된 셀(`fill_type`이 존재하고 `none`이 아닌 경우)에 대해 `copy` 복제를 적용하여 로드맵에 색상을 완벽하게 이식하는 코드를 추가했습니다.

### ④ 최종 추출 엑셀 파일명 포맷 표준화 (`app.py`)
* **요구 분석**: 추출된 로드맵 엑셀의 파일명을 사용자가 손쉽게 식별하고 보관할 수 있도록 특정 규칙의 명명 표준을 반영해야 했습니다.
* **구현 내용**:
  - 기존 YYYYMMDD 접두어 기반의 단순 명명 규칙 대신, `{올해}년 OO그룹_{기업명} 정보보안감사_보안솔루션로드맵_{생성일}.xlsx` 형태의 포맷 규칙을 적용했습니다.
  - 서버 시각을 파싱해 올해 년도(YYYY) 및 오늘 일자(YYYYMMDD)를 추출하고, RAG로 매핑된 고객사 기업명(`{company}`)과 조합하여 `2026년 OO그룹_OO기업명 정보보안감사_보안솔루션로드맵_20260601.xlsx` 형태의 표준 파일명으로 안전하게 브라우저에 리턴하도록 반영했습니다.

### ④ 3-Column Premium Grid 드롭존 UI/UX 리디자인
* **요구 분석**: 사용자가 3가지 자료를 한눈에 보며 편리하게 개별 업로드할 수 있는 모던하고 가시성 높은 인터페이스가 요구되었습니다.
* **구현 내용**:
  - `templates/index.html` 의 기존 1개 드롭존 카드를 확장하여 가로 배치형의 반응형 **3-Column Grid 드롭존** 레이아웃으로 전면 리디자인했습니다.
  - 각 파일의 특성에 맞추어 FontAwesome의 고유 아이콘(`상세체크리스트 - 청록 배지`, `사전환경조사서 - 퍼플 배지`, `자산목록 - - 레드 배지`)과 업로드 성공 시의 파일명 각인 및 `(필수)`, `(선택)` 태그를 명확하게 적용하여 프리미엄 테마에 매칭했습니다.
  - `static/css/style.css` 에 퍼플 및 레드 드롭존 호버/드래그오버 전용 액센트 스타일링(`rgba` 색상 및 입체적인 내부 glow shadow)을 구축하여 뛰어난 시각적 상호작용 피드백을 완성했습니다.

### ⑤ 중복 클릭/전송 가드 전역 적용
* **개선 사항**: 업로드나 AI 분석 기동 등 비동기 요청이 완료되지 않은 상태에서 사용자가 버튼을 더블클릭하거나 연타할 시 동일 요청이 중복 발송되어 Ollama 서버 및 백엔드 스레드가 동시 점유되는 장애 요인을 차단했습니다.
* **적용 내역**:
  - 파일 인풋 변경 시 로딩 상태 동안 해당 인풋 엘리먼트(`fileInput`, `fileInputEnv`, `fileInputAsset`)를 즉시 비활성화(`disabled = true`) 처리하고 비동기 통신 완료 후 복구하도록 가드를 구현했습니다.
  - AI 매핑 기동 및 최종 엑셀 추출 버튼에도 전송 시작 시 즉시 버튼 비활성화 및 흐릿한 시각 효과(`opacity: 0.6`)를 제공하여 물리적인 연타 및 중복 처리를 완벽하게 차단했습니다.

### ⑥ 프론트엔드 '적용됨' 상태 영구 고정 및 실시간 메모리 동기화
* **개선 사항**: 위험도, 예상 예산, 로드맵 연도 등의 매핑 결과 세부 사항을 수정할 때, 기존의 '입력 완료' 버튼이 '적용됨'으로 변경되었다가 연동된 인풋의 수정에 의해 수시로 '입력 완료'로 다시 풀려버리는 사용성 불편을 제거했습니다.
* **적용 내역**:
  - 결과 테이블 상의 입력 변경 시 버튼을 '입력 완료' 상태로 초기화하는 `resetStatus` 리스너를 완전히 폐지했습니다.
  - 대신, 사용자가 각 필드값을 수정하면 실시간 비동기 이벤트 리스너(`change`, `input`)가 백그라운드 자바스크립트 메모리 배열(`mappedResults`)에 즉시 변경된 최신 상태를 자동 동기화하도록 고도화하여 데이터 무손실성을 확보했습니다.

### ① AI 매핑 결과 N/A 매핑 누락 문제 해결 (`roadmap_agent_llm_new.py`)
* **현상**: AI 분석 기동 후 추천 보안 솔루션(제조사) 및 법적 요구사항이 UI 상에서 N/A로 고정되어 출력되던 오류가 있었습니다.

* **해결**:

  - LLM 호출 직후 RAG 매핑 엔진이 감지한 실제 자사 솔루션의 제품명과 제조사를 최종 `final_res` 객체의 `"추천솔루션"` 및 `"제조사"`에 명시적으로 오버라이딩하여 자동 바인딩해주는 **후처리 안전 매핑 로직**을 추가함으로써 UI 상의 누락 문제를 원천 차단했습니다.

### ② 최종 엑셀 자산 목록 포맷 무결성 및 셀 배경색 완벽 이식
* **현상**: 최종 추출된 엑셀 파일 내 자산 영역의 행 높이가 비정상적으로 왜곡되고, 템플릿의 원래 양식인 '비고(I~P열 가로 병합)'가 깨지며 기밀성/무결성 등의 수치가 노출되어 템플릿 규격이 손상되었습니다. 또한, 원본 자산목록에 칠해져 있던 셀 배경색들이 이식되지 않는 문제가 있었습니다.
* **해결**:
  - **포맷 구조 복원**: 기밀성/무결성 등의 등급 데이터를 억지로 쪼개어 기입하던 방식을 배제하고, 원래 템플릿의 수려한 격자 형태인 **비고 영역(I열~P열 가로 병합 상태)의 가로 합치기를 매 자산 행마다 완벽히 재구축**하여 템플릿의 양식과 무결성을 그대로 유지했습니다.
  - **자산 셀 배경색 이식**: 자산목록 평가서 파싱 시 개별 셀들(구분, IP, 벤더, 기능 등)의 `fill`(PatternFill) 속성을 정확히 추출하고, 최종 엑셀 쓰기 시 각 데이터 셀에 `copy` 복제하여 **원본의 채우기 배경색을 완벽하게 이식**했습니다.
  - **테두리 및 높이 왜곡 방지**: 행 삽입 시 타이틀 행(`roadmap_title_row`)으로부터 굵은 테두리 서식이 새로 생성된 자산 행들에 상속되어 경계선이 굵어지던 현상을 방지하기 위해, 삽입된 자산 행 전체 열(1~16열)에 대해 스타일 복사 후 **얇은 격자 테두리(`thin_border`)를 일괄 적용**했습니다. 아울러 삽입 행의 높이를 템플릿의 규격 표준 높이(23.25)로 정확히 복사 동화시켜 비대화 현상을 차단했습니다.

## 6. 사전환경조사서 병합 셀 파싱 및 자산 중요도 기입/정렬 고도화 (2026.06.01 추가 보완)

### ① 사전환경조사서 병합 셀 파싱 안정성 개선
* **현상 및 원인**: 사전환경조사서 내 신청업체 정보 영역이 B~F열 또는 H~K열 등으로 병합된 경우, openpyxl의 `data_only=True` 호출 특성에 의해 병합 범위 내의 첫 번째 셀이 아닌 다른 셀에 데이터가 남아있거나 셀 분할 상태에 따라 데이터 조회가 실패하는 결함이 있었습니다.
* **해결**:
  - 지정한 열 범위 내에서 빈 값이 아닌 최초의 유효 데이터를 탐색하는 `get_merged_cell_value` 헬퍼 함수를 구축했습니다.
  - 이를 사전환경조사서 파싱부의 기업명, 사업형태, 임직원수, 주요적용법률(B~F열, 2~6열) 및 사업분야, 정보통신여부, 매출, 감독기관(H~K열, 8~11열) 영역에 전면 도입하여 어떠한 병합 서식 하에서도 신청업체 데이터가 최종 로드맵 엑셀 상단(B5~B8, I5~I8)에 정확히 기입되도록 안정성을 극대화했습니다.

### ② 보안 솔루션 세부 현황 내 중요도 평가지표 매핑 및 스타일 고도화
* **요구 분석**: 기존의 로드맵 엑셀 자산 목록 테이블 비고란(I~P열)을 `기밀성 | 무결성 | 가용성 | 합계 | 등급`으로 쪼개서 표현하고, 셀 값들을 가로/세로 가운데 정렬(중앙 맞춤)로 지정하며, 원본 엑셀에 칠해져 있던 모든 색상과 테두리가 누락 없이 전파되도록 개선해야 했습니다.
* **해결**:
  - **헤더 영역 리포맷 및 스타일 복사**: 로드맵 엑셀 12행 헤더 영역의 I~P열 병합을 개별 컬럼으로 쪼개어 `기밀성`(Col 9), `무결성`(Col 10), `가용성`(Col 11), `합계`(Col 12), `등급`(Col 13), `비고`(Col 14~16)를 정의하였으며, 8열(담당부서)의 스타일을 기밀성~등급 및 비고 병합 영역 전체에 고루 복사 적용하고 가운데 정렬을 지정했습니다.
  - **자산 중요도 지표 배경색 완벽 이식**: 자산목록 원본 시트 파싱 시 기밀성, 무결성, 가용성, 합계, 등급의 `PatternFill`(배경색) 속성을 추가로 수집하여 로드맵 기입 시점에 각 셀로 정확히 이식했습니다.
  - **비고 병합 영역 채우기 통일**: 비고란(Col 14~16 병합)의 경우 병합되는 전체 3개 셀에 동일한 배경색과 얇은 격자 테두리(`thin_border`)를 일괄 적용한 뒤 `merge_cells`를 수행함으로써 레이아웃 깨짐을 방지하고 Premium 테마 서식을 유지했습니다.
  - **일괄 가운데 정렬(중앙 맞춤) 적용**: 기밀성~등급 및 비고 병합란을 포함한 9~16열(I열~P열)의 자산 데이터 셀에 대해 `Alignment(horizontal="center", vertical="center", wrap_text=True)`를 강제 부여하여 단정하고 수려한 포맷을 완성했습니다.
* **파이프라인 검증 결과**:
  - `scratch/test_pipeline.py` 스크립트를 통해 임시 테스트를 수행한 결과, 사전환경조사서 병합 데이터(기업명: OO그룹 (가족사명), 사업분야) 추출 성공, 12행 헤더 개편 확인, 방화벽 행 구분 및 기밀성 데이터 셀에 빨간색 배경(`00FF0000`) 복사 확인, 기밀성~등급 및 비고 셀의 `center` 정렬이 정상 작동함을 입증하였습니다.
