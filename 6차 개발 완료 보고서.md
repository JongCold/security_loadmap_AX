# 6차 개발 완료 보고서 (엑셀 추출 가시성 최적화 및 비고란 포맷팅 일치화)

본 보고서는 **AX 기반 정보보안감사 보안솔루션 로드맵 자동 수립 시스템**의 6차 고도화 단계에서 진행된 최종 엑셀 추출 결과물의 동적 가시성 최적화, 개선방안 가독성 극대화 및 비고란 포맷 통일 엔진 구현 결과를 명세합니다.

---

## 1. 개발 개요

* **목표**:
  - 기존 템플릿의 고정된 행 높이로 인해 AI가 출력한 긴 텍스트(세부점검내용, 운영실태, 개선방안, 비고 등)가 잘리고 겹쳐 보이던 가시성 결함을 완전 해결.
  - 가변 줄바꿈(`\n`) 및 글자 수에 실시간 반응하여 행 높이를 동적 산정하는 가변 행 높이 조절 알고리즘 탑재.
  - 가로 너비 부족으로 한 글자씩 밑으로 접히는 문제를 해결하기 위한 열별 최적 너비 강제 적용.
  - 엑셀 '개선방안' 셀의 글자 색상을 빨간색(`FF0000`)으로 일괄 강조하여 주목성 확보.
  - '비고' 셀의 글자 색상을 검정색(`000000`)으로 통일하고, `[추천 솔루션] : {제품명}|{제조사}` 및 번호 매김 형식의 `[선정 이유]` 조합으로 강제 포맷팅하는 리포맷 엔진 개발.
  - 모든 결과 셀에 대해 최적화된 가로/세로 정렬 배분 및 자동 줄바꿈(`wrap_text=True`) 적용.
* **수행 기간**: 2026년 6월 2일
* **핵심 구현체**:
  - `roadmap_agent_llm_new.py` (`reformat_note` 함수 신설, `generate_roadmap_excel` 내 자산목록/로드맵 동적 높이 계산, 열 너비 일괄 최적화, 폰트 색상/정렬 강제화 코드 통합)

## 2. 6차 개발 핵심 항목 및 구현 내용

### ① 가변 행 높이 동적 산정 알고리즘 (`count_lines` 내장 모듈)
* **요구 사항**: 엑셀 셀 내에 긴 줄글이 들어갈 때 행 높이가 자동으로 늘어나지 않아 수동으로 행을 벌려야 했던 가독성 저하를 해결해야 했습니다.
* **구현 내용**:
  - 입력되는 텍스트의 가시 줄 수를 정밀 예측하는 `count_lines(text, max_chars)` 로컬 함수를 설계했습니다.
  - 한글 1글자(너비 2), 영문/숫자/공백 1글자(너비 1) 기준의 멀티바이트 특성을 고려하여 가상 비주얼 텍스트 너비를 계산하고, 이를 지정된 열의 최대 글자 수용량으로 나누어 실제 필요한 줄 수를 예측합니다.
  - 로드맵 결과 행의 `세부점검내용`, `운영현황_증적`, `개선방안`, `비고` 영역 중 가장 긴 줄 수(`max_lines`)를 판별하여 행 높이를 동적 계산합니다:
    $$\text{Calculated Height} = \max(35, \text{Max Lines} \times 16 + 12)$$
  - 자산목록 데이터 기입부에도 기능명과 비고 내용의 길이를 측정해 행 높이를 가변 조절(`max(28, max_asset_lines * 16 + 8)`)하도록 탑재하여 레이아웃 답답함을 근본적으로 타개했습니다.

### ② 열 너비 일괄 최적화 (가로 가시성 확장)
* **요구 사항**: 템플릿의 가로 폭이 협소하여 긴 제품명이나 법적 요구사항, 비고란이 수직으로 기괴하게 접히던 문제를 해결해야 했습니다.
* **구현 내용**:
  - 엑셀 통합 저장 직전에 A열부터 P열까지의 너비를 사용자가 보기에 가장 쾌적한 전용 표준 스펙으로 재조정하는 `column_widths` 매핑 테이블을 구현했습니다.
  - 특히 텍스트가 조밀하게 들어가는 **비고 열(P열)의 너비를 45로 과감하게 넓혔으며**, **과제명(J열)은 28**, **법적요구(K열)는 22** 등으로 확보하여 가로 가독성을 프리미엄 등급으로 도약시켰습니다.

### ③ 개선방안 빨간색 강조 및 비고란 검정색화
* **요구 사항**: 최종 로드맵 장표에서 조치가 시급한 개선방안은 빨간색으로 시선을 끌고, 비고란은 신뢰감 높은 검정색으로 단정하게 표시해야 했습니다.
* **구현 내용**:
  - `generate_roadmap_excel` 루프에서 개선방안 데이터가 쓰이는 **7, 8열**의 폰트 속성 중 색상을 빨간색(`FF0000`)으로 명시적 변경했습니다.
  - 비고란이 작성되는 **16열**의 폰트는 검정색(`000000`)으로 강제 변경하여 일괄 통일감을 이끌어 냈습니다.
  - 기존 템플릿 스타일을 손상하지 않기 위해 원본 폰트 패밀리(글꼴명, 굵기, 크기 등)는 완벽히 상속 복제(`Font(...)` 생성)한 채로 색상 메타데이터만 타겟팅 교체했습니다.

### ④ 비고란 프리미엄 리포맷 엔진 (`reformat_note`)
* **요구 사항**: 분산되거나 일관성 없이 저장된 AI의 비고 추천 데이터를 아래의 정형화된 가이드 규격으로 일괄 보정해야 했습니다.
  ```text
  [추천 솔루션] : {제품명}|{제조사}
  [선정 이유]
  1. 이유 1
  2. 이유 2
  ```
* **구현 내용**:
  - 정규식을 활용하여 기존 비고 데이터 내 `[추천 솔루션]`과 `[선정 이유]` 텍스트 영역을 분할 파싱하는 `reformat_note` 범용 전처리기를 구현했습니다.
  - 제조사 - 제품명 순서를 판별하여 유저가 요청한 `{제품명}|{제조사}` 형태로 완벽하게 재조정합니다 (솔루션이 없을 시 `[추천 솔루션] : N/A` 로 정돈).
  - 선정 이유 뒤의 설명 텍스트를 마침표(`.`) 기준으로 슬라이싱하고 각 문장들의 불필요한 번호 표기를 제거한 뒤, `1. 이유1`, `2. 이유2`의 형태로 최대 3개 문장까지 순서대로 정렬 기입합니다.
  - 선정 이유가 1문장 이하로 요약 가능한 단문인 경우, 굳이 줄을 늘리지 않고 1번 문장 단독 형태로 깔끔하게 마감하도록 예외 가드를 설계했습니다.

### ⑤ 전 데이터 셀 가운데 정렬 및 자동 줄바꿈 강제
* **구현 내용**:
  - 스타일만 단순 복사할 경우 자동 줄바꿈(`wrap_text`) 속성이 누락되어 표 밖으로 글자가 튀어나가는 현상을 차단하기 위해, 로드맵 표의 전 컬럼(1~16열)에 대해 강제로 `wrap_text=True`를 부여했습니다.
  - 또한 열의 역할에 맞추어 항목명/영역/법률/연도는 `center`(가운데), 세부내용/실태/개선방안/비고는 `left`(왼쪽)로 조율하는 가로 정렬 매트릭스를 강제 바인딩하여 엑셀 가독성을 극대화했습니다.
