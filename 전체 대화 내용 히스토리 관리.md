# 전체 대화 내용 히스토리 관리 문서

본 문서는 **AX (AI Transformation) 기반 정보보안감사 보안솔루션 로드맵 자동 수립 시스템**의 개발 기획부터 최종 6차 가시성 최적화 단계까지의 모든 대화 흐름, 현업 애로사항, 직면했던 다양한 기술적 결함 및 버그 해결 이력을 날짜순으로 총괄 기술하여 향후 유지보수와 기능 확장의 참고 문서로 활용할 수 있도록 합니다.

## 2. 차수별 상세 대화 맥락 및 트러블슈팅 이력

### 2.1 1차 & 2차 개발 단계 (기초 수립 및 서식 복제)
- **사용자 Pain Point**: "엑셀 로드맵 파일을 사람이 일일이 만드는데 시간이 너무 걸립니다. 기존의 양식 선 굵기, 색상, 글꼴이 하나도 깨지지 않으면서 취약점으로 평가된 항목에 자사 보안 제품을 자동으로 매핑해서 엑셀을 뽑아주세요."
- **주요 해결 과정**:
  - openpyxl로 새로운 셀을 생성할 때 테두리, 배경색, 폰트 등의 속성이 파괴되는 문제를 해결하기 위해, 원래 템플릿의 스타일 인스턴스를 깊은 복사(`copy`)하여 대상 셀에 씌우는 **Style Cloner** 구조를 최초 구현했습니다.
  - 평가 시트에서 개선 조치가 필요한 취약 사항을 빨간색 글씨(ARGB: `FFFF0000`)로 표기하는 규칙을 감별하는 매커니즘을 적용했습니다.

### 2.2 3차 개발 단계 (폐쇄망 AI 엔진 구축 및 RAG 고도화)
- **사용자 Pain Point**: "고객사의 내부 진단 데이터를 외부 클라우드 AI(OpenAI 등)에 보내는 것은 기밀 유출 규정 위반입니다. 완전 오프라인 내부망에서 돌 수 있게 해주시고, RAG 매핑 정확도를 올려주세요. 방화벽 결함에 자꾸 엉뚱한 문서 보안 솔루션이 추천됩니다."
- **주요 해결 과정**:
  - 로컬 PC에서 단독 기동하는 sLLM인 **Ollama Gemma 2:2b**를 백엔드에 장착하여 기밀 데이터 유출 위험을 100% 차단했습니다.
  - 임베딩 거리가 협소한 한글 보안 고유 명사들("방화벽", "이차 인증" 등)의 정확한 구분을 위해, 벡터 유사도(30%)에 더해 솔루션 DB 명세에 포함된 주요 명사 키워드의 빈도를 스캔하여 가중치(70%)를 연산하는 **시맨틱 Re-ranking 알고리즘**을 개발하여 정확한 매핑을 구현했습니다.

### 2.3 4차 개발 단계 (프론트엔드 대시보드 리디자인 및 셀 병합 무결성 확보)
- **사용자 Pain Point**: "진단 파일 여러 개를 업로드하기 편하게 UI를 다듬어주시고, 엑셀을 다운로드할 때 브라우저가 위험한 파일이라며 경고를 내는 현상을 해결해주세요. 그리고 매핑 후 자산 수가 변경될 때 엑셀 하단 로드맵 테이블의 셀 병합이 다 깨져서 틀어집니다."
- **주요 해결 과정**:
  - Javascript Fetch Blob 방식 다운로드가 브라우저의 Untrusted Content 필터에 차단되던 것을 우회하고자, 동적으로 HTML 폼을 만들어 서브밋하는 **네이티브 Form Submit POST 방식**으로 다운로드 흐름을 교체하고 보안 헤더들을 강화했습니다.
  - 가변 행 삽입/삭제 시 셀 병합 정보가 깨지던 오류를 막기 위해, 엑셀 가공 전 원본의 병합 셀 영역 정보(`merged_cells.ranges`)를 모두 가상의 큐에 백업한 후 전체 표를 언머지하고, 행 크기 가변 연산이 끝난 시점에 늘어난 행의 오프셋(`diff` 및 `roadmap_diff`)만큼 행 인덱스를 더해 최종 1회 일괄 재병합을 진행하는 **통합 오프셋 병합 무결성 알고리즘**을 도입했습니다.

### 2.4 5차 개발 단계 (사전환경조사서 파싱 및 자산 현황 연동)
- **사용자 Pain Point**: "사전환경조사서 엑셀에서 업체 정보를 파싱해서 최종 로드맵 상단에 꽂아주고, 자산 목록 파일의 보안 장비 현황을 로드맵 밑의 자산 세부 현황에 그대로 합쳐주세요. 기밀성/무결성 등의 수치 점수도 쪼개어 정렬하고 배경색도 살려주세요."
- **주요 해결 과정**:
  - 사전조사서의 기업명 등 정보 셀이 B~F열 등으로 다양하게 가로 병합된 환경에 대처하기 위해 범위 내 유효 데이터를 순회하는 `get_merged_cell_value` 함수를 개발했습니다.
  - 자산목록 중 기가동 정지 상태인 자산(빨간 배경색)을 `is_red_fill` 판단기로 식별해 비고란에 `운영하고 있지 않는 자산` 문구를 자동 주입했습니다.
  - 로드맵 자산 테이블의 기밀성, 무결성, 가용성, 합계(SUM 수식), 등급, 비고란을 단독 컬럼으로 분리하고, 원본 자산 셀의 채우기 배경색을 최종 로드맵 셀로 복제 이식한 뒤 전체 영역에 `Alignment(horizontal="center", vertical="center", wrap_text=True)` 정렬을 부여했습니다.

### 2.5 6차 개발 단계 (엑셀 가시성 극대화 및 백엔드 PNA/타임아웃 우회)
- **사용자 Pain Point**: "추출된 엑셀을 열어보면 행 높이가 너무 낮아서 글씨가 다 잘려 보여 수동으로 고쳐야 합니다. 개선방안은 빨간색으로, 비고란은 검정색으로 하고 비고 포맷도 `[추천 솔루션] : 제품명|제조사` 형태로 맞춰주세요. 추가로 Vercel 배포 후 외부에서 접속하면 백엔드 서버 연결이 아예 안 되거나 타임아웃 에러가 납니다."
- **주요 해결 과정**:
  - **가변 행 높이 조절**: 멀티바이트 길이와 줄바꿈 개수를 계산하는 `count_lines` 로컬 헬퍼를 추가하여 행별 표준 높이를 동적 배정하고, 열 너비를 일괄 확장(`column_widths`)하여 글자 접힘 문제를 원천 차단했습니다.
  - **색상 및 비고 포맷 통일**: 개선방안(7, 8열)은 빨간색 폰트, 비고(16열)는 검정색 폰트로 상속 변경하고, `reformat_note` 변환기를 장착하여 비고 데이터를 사용자 요구 규격 및 번호 순서가 매겨진 선정 이유 리스트 형태로 후처리 통일했습니다.
  - **PNA CORS 우회**: Vercel(HTTPS)과 로컬 사설망 백엔드 통신 시 크롬의 사설망 통제 정책(PNA)을 통과하도록, 응답 사설망 허용 헤더의 다중 삽입 오류를 청소하고 단일 `"true"` 문자열로만 응답하게 보정하는 **`PNAMiddleware` WSGI 미들웨어**를 최상단 레이어에 부착했습니다.
  - **LocalTunnel 504 Timeout 방지**: LocalTunnel 게이트웨이의 60초 타임아웃 제한을 우회하기 위해, 매핑 API `/api/map`을 **백그라운드 스레드 비비동기 방식**으로 전환하고, 클라이언트가 `/api/map/status/<task_id>`를 1.5초마다 호출하는 **실시간 폴링 체계**를 프론트엔드와 백엔드에 구축하여 긴 AI 분석 시간 지연을 완벽하게 극복했습니다.

## 3. 종합 요약 및 시스템 아키텍처 상태

본 정보보안감사 로드맵 시스템은 1~6차에 걸친 정교한 고도화 피드백을 통해 오프라인 sLLM 인공지능 매핑 기술과 openpyxl 기반 스타일 보존 파이프라인의 완성도를 결합한 완성형 엔터프라이즈 자동화 솔루션입니다. 기밀이 유지되는 독립된 로컬망 환경을 견고하게 보장하면서도, Vercel 및 LocalTunnel 터널링 연동을 통해 외부 이해관계자들과 안전하고 즉각적인 사용성 테스트 환경을 유연하게 공유할 수 있는 아키텍처 무결성을 확보했습니다.
