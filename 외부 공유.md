# 외부 공유 (Vercel 배포) 히스토리 관리 문서

본 문서는 **Security Roadmap Agent** 시스템의 UI를 외부 사용자에게 공유하기 위해 **Vercel**을 통해 정적 사이트로 배포한 전 과정을 기록합니다.

---

## 1. 배포 아키텍처 개요

### 1.1 왜 Vercel을 선택했는가?
- **외부 사용자 사용성 테스트 (UX Test)** 를 빠르게 진행하기 위해 Vercel의 정적 호스팅 서비스를 채택
- GitHub 리포지토리 연동만으로 자동 배포(CI/CD)가 가능
- 무료 플랜에서도 HTTPS, CDN 지원

### 1.2 배포 구조

```
┌─────────────────────────────────┐     ┌────────────────────────────────┐
│  Vercel (정적 호스팅)            │     │  로컬 서버 (내부 운영)           │
│  ──────────────────             │     │  ───────────────               │
│  public/                        │     │  app.py (Flask API)            │
│  ├── index.html                 │     │  roadmap_agent_llm_new.py      │
│  ├── css/style.css              │     │  rag_engine.py                 │
│  └── js/app.js                  │     │  Ollama + ChromaDB             │
│                                 │     │  openpyxl Excel Engine         │
│  ✅ 백엔드 주소 동적 설정 제어   │     │  ✅ 실제 AI 매핑 + 엑셀 추출     │
└─────────────────────────────────┘     └────────────────────────────────┘
```

> ⚠️ **중요**: Vercel에 배포되는 것은 **프론트엔드(UI) 전용**입니다. 실제 AI 매핑, 엑셀 추출, RAG 벡터 검색 등의 백엔드 기능은 로컬 Flask 서버가 필요합니다.

### 1.3 브라우저 Mixed Content 제한 및 해결 방안
- Vercel은 보안 연결(**HTTPS**)을 강제하는 반면, 로컬 Flask 백엔드는 기본적으로 일반 연결(**HTTP**)을 사용합니다.
- 이에 따라 크롬 등 최신 브라우저는 **HTTPS 페이지에서 HTTP 서버로의 통신(localhost 포함)을 보안상 차단**합니다.
- 이 문제를 방지하기 위해 사용자는 화면 상단의 **연결 제어 패널**을 통해 직접 백엔드 주소를 변경할 수 있으며, 최선의 해결책은 **로컬 호스트로 직접 접속(http://localhost:5000)** 하는 것입니다.

## 2. Vercel 배포 절차 (Step-by-Step)

### Step 1: GitHub 리포지토리 준비
```bash
# 프로젝트 디렉터리에서 Git 커밋
cd "Security loadmap_Auto"
git add vercel.json public/
git commit -m "feat: Vercel 외부 배포용 정적 프론트엔드 추가 및 데모 제거"
git push origin main
```

### Step 2: Vercel 프로젝트 생성
1. [vercel.com](https://vercel.com) 에 접속하여 GitHub 계정으로 로그인
2. **"Add New Project"** 클릭
3. GitHub 리포지토리 목록에서 `security_loadmap_AX` 선택
4. **Framework Preset**: `Other` 선택 (정적 사이트)
5. **Root Directory**: 기본값 유지 (프로젝트 루트)
6. **Build & Output Settings**:
   - Build Command: 비워둠 (빌드 불필요)
   - Output Directory: `public` (vercel.json에서 자동 인식)
7. **Deploy** 클릭

### Step 3: 배포 URL 확인
- 배포 완료 후 Vercel이 자동으로 URL을 부여합니다
- 형식: `https://security-roadmap-agent-xxxxx.vercel.app`
- 이 URL을 외부 사용자에게 공유하여 사용성 테스트를 진행합니다

### Step 4: 자동 배포 (CI/CD)
- GitHub `main` 브랜치에 Push할 때마다 Vercel이 자동으로 재배포합니다
- PR(Pull Request) 생성 시 Preview 배포도 자동 생성됩니다

## 4. 보안 유의사항

### 4.1 마스킹 처리 완료 항목
- ✅ `.env` 파일: `.gitignore`에 등록되어 Git에 업로드되지 않음
- ✅ `엑셀 파일(.xlsx)`: `.gitignore`에 등록되어 Git에 업로드되지 않음
- ✅ `uploads/` 디렉터리: `.gitignore`에 등록되어 Git에 업로드되지 않음
- ✅ `chroma_db/`: `.gitignore`에 등록되어 Git에 업로드되지 않음

### 4.2 Vercel 환경에서의 데이터 보안
- Vercel에 배포된 정적 사이트에는 **백엔드 코드, 엑셀 데이터, RAG 벡터 DB** 가 포함되지 않음
- 모든 민감 데이터는 로컬 서버에서만 처리됨

## 5. 로컬 서버 연동 방법 (해결책)

Vercel에 배포된 프론트엔드를 로컬 Flask 서버와 연동하여 사용하려 할 때, 브라우저가 연결을 차단하는 현상은 아래 두 방법으로 해결할 수 있습니다.

### 방법 A (강력 추천): 로컬 URL 직접 접속
로컬 Flask 서버가 켜져 있는 상태에서 Vercel 대신, 브라우저 주소창에 다음 주소로 접속해 사용합니다.
- **주소**: [http://localhost:5000](http://localhost:5000)
- **효과**: 브라우저 보안 제약(Mixed Content)이 발동하지 않으므로 추가 설정 없이 바로 완벽하게 100% 로컬 백엔드 기능이 동작합니다.

### 방법 B: HTTPS 터널링 주소 설정
외부 주소(Vercel)를 꼭 유지해야 하는 경우, 로컬 서버를 임시 HTTPS 주소로 포워딩합니다.
1. 터미널에서 `ngrok` 등을 실행합니다.
   ```bash
   ngrok http 5000
   ```
2. 발급되는 HTTPS 주소(예: `https://abcd-123.ngrok-free.app`)를 복사합니다.
3. Vercel 사이트 상단의 **"백엔드 서버 연결 제어"** 입력창에 복사한 주소를 입력하고 **"연결하기"** 버튼을 누릅니다.
4. 연결 성공 후 브라우저 저장소에 주소가 기억되므로 원활하게 사용이 가능합니다.
