# 외부 공유 (Vercel 배포) 히스토리 관리 문서

본 문서는 **Security Roadmap Agent** 시스템의 UI를 외부 사용자에게 공유하기 위해 **Vercel**을 통해 정적 사이트로 배포한 전 과정을 기록합니다.

---

## 1. 배포 아키텍처 개요

### 1.1 왜 Vercel을 선택했는가?
- **외부 사용자 사용성 테스트 (UX Test)** 를 빠르게 진행하기 위해 Vercel의 정적 호스팅 서비스를 채택
- GitHub 리포지토리 연동만으로 자동 배포(CI/CD)가 가능
- 무료 플랜에서도 HTTPS, CDN 지원

### 1.2 배포 구조

```
┌─────────────────────────────────┐     ┌────────────────────────────────┐
│  Vercel (정적 호스팅)            │     │  로컬 서버 (내부 운영)           │
│  ──────────────────             │     │  ───────────────               │
│  public/                        │     │  app.py (Flask API)            │
│  ├── index.html                 │     │  roadmap_agent_llm_new.py      │
│  ├── css/style.css              │     │  rag_engine.py                 │
│  └── js/app.js                  │     │  Ollama + ChromaDB             │
│                                 │     │  openpyxl Excel Engine         │
│  ✅ 백엔드 주소 동적 설정 제어   │     │  ✅ 실제 AI 매핑 + 엑셀 추출     │
└─────────────────────────────────┘     └────────────────────────────────┘
```

> ⚠️ **중요**: Vercel에 배포되는 것은 **프론트엔드(UI) 전용**입니다. 실제 AI 매핑, 엑셀 추출, RAG 벡터 검색 등의 백엔드 기능은 로컬 Flask 서버가 필요합니다.

### 1.3 브라우저 Mixed Content 제한 및 해결 방안
- Vercel은 보안 연결(**HTTPS**)을 강제하는 반면, 로컬 Flask 백엔드는 기본적으로 일반 연결(**HTTP**)을 사용합니다.
- 이에 따라 크롬 등 최신 브라우저는 **HTTPS 페이지에서 HTTP 서버로의 통신(localhost 포함)을 보안상 차단**합니다.
- 이 문제를 방지하기 위해 사용자는 화면 상단의 **연결 제어 패널**을 통해 직접 백엔드 주소를 변경할 수 있으며, 최선의 해결책은 **로컬 호스트로 직접 접속(http://localhost:5000)** 하는 것입니다.

## 2. Vercel 배포 절차 (Step-by-Step)

### Step 1: GitHub 리포지토리 준비
```bash
# 프로젝트 디렉터리에서 Git 커밋
cd "Security loadmap_Auto"
git add vercel.json public/
git commit -m "feat: Vercel 외부 배포용 정적 프론트엔드 추가 및 데모 제거"
git push origin main
```

### Step 2: Vercel 프로젝트 생성
1. [vercel.com](https://vercel.com) 에 접속하여 GitHub 계정으로 로그인
2. **"Add New Project"** 클릭
3. GitHub 리포지토리 목록에서 `security_loadmap_AX` 선택
4. **Framework Preset**: `Other` 선택 (정적 사이트)
5. **Root Directory**: 기본값 유지 (프로젝트 루트)
6. **Build & Output Settings**:
   - Build Command: 비워둠 (빌드 불필요)
 ## 5. 로컬 서버 연동 방법 (해결책)

로컬 Flask 백엔드가 임시 SSL 인증서(`ssl_context='adhoc'`) 기반의 **HTTPS(https://localhost:5000)**로 구동되므로, Vercel 정적 사이트(HTTPS)와의 연결 시 브라우저 차단 문제를 우회하여 안전하게 연동할 수 있습니다.

### 방법 A (추천): adhoc HTTPS 직접 연결 및 브라우저 보안 승인
로컬 Flask 백엔드가 켜져 있는 상태에서 Vercel 앱과의 통신을 활성화하는 가장 간편한 방법입니다.

1. **최초 브라우저 보안 승인**:
   - 새 웹 브라우저 탭을 열고 [https://localhost:5000/api/rag/status](https://localhost:5000/api/rag/status) 주소로 직접 접속합니다.
   - 자체 서명된(adhoc) 인증서를 사용하므로 경고 화면이 표시됩니다.
   - **"고급"** 또는 **"상세 정보 보기"** 버튼을 클릭한 뒤, **"localhost(안전하지 않음)로 이동"** 링크를 눌러 접속을 허용합니다. (JSON 상태 메시지가 화면에 출력되면 정상 승인 완료)
2. **연동 수행**:
   - Vercel 호스팅 사이트(`https://security-loadmap-ax.vercel.app`)로 돌아옵니다.
   - 상단 **"백엔드 서버 연결 제어"** 패널에 `https://localhost:5000`이 지정된 것을 확인한 후 **"연결하기"** 버튼을 누릅니다.
   - 브라우저 로컬 세션에서 백엔드가 정상적으로 인식되어 외부 인터넷 경유 없이 다이렉트로 안전하게 연동됩니다.

### 방법 B: HTTPS 터널링 주소 설정 (LocalTunnel 또는 ngrok)
외부의 다른 팀원이나 사용자에게 로컬 백엔드를 포함하여 전체 테스트 환경을 완벽히 공유해야 하는 경우 사용합니다. LocalTunnel은 별도의 가입이나 인증 토큰 없이 간편하게 사용할 수 있어 권장합니다.

#### Option 1: LocalTunnel 사용
1. 로컬에 Node.js가 설치되어 있어야 합니다. 터미널에서 다음 명령어를 실행합니다.
   ```bash
   npx localtunnel --port 5000 --local-https
   ```
   또는 특정 서브도메인을 지정하여 고정하고 싶다면:
   ```bash
   npx localtunnel --port 5000 --local-https --subdomain security-roadmap-backend
   ```
2. 발급되는 HTTPS 주소(예: `https://security-roadmap-backend.localthrough.me` 등)를 복사합니다.
3. 복사한 외부 터널 주소에 최초 접속 시, 안내 화면에서 **"Click to Continue"** 버튼을 눌러 터널 접속을 승인해 줍니다.

#### Option 2: ngrok 사용
1. 터미널에서 `ngrok`을 실행하여 로컬 HTTPS 포트를 포워딩합니다.
   ```bash
   ngrok http https://localhost:5000
   ```
2. 발급되는 HTTPS 주소(예: `https://abcd-123.ngrok-free.app`)를 복사합니다.

#### 연결 및 적용 방법
1. Vercel 사이트 상단의 **"백엔드 서버 연결 제어"** 입력창에 복사한 터널링 HTTPS 주소를 입력하고 **"연결하기"** 버튼을 누릅니다.
2. 연결 성공 후 브라우저 로컬 저장소에 주소가 기억되므로 원활하게 사용이 가능합니다.

## 5. 로컬 서버 연동 방법 (해결책)

Vercel에 배포된 프론트엔드를 로컬 Flask 서버와 연동하여 사용하려 할 때, 브라우저가 연결을 차단하는 현상은 아래 두 방법으로 해결할 수 있습니다.

### 방법 A (강력 추천): 로컬 URL 직접 접속
로컬 Flask 서버가 켜져 있는 상태에서 Vercel 대신, 브라우저 주소창에 다음 주소로 접속해 사용합니다.
- **주소**: [http://localhost:5000](http://localhost:5000)
- **효과**: 브라우저 보안 제약(Mixed Content)이 발동하지 않으므로 추가 설정 없이 바로 완벽하게 100% 로컬 백엔드 기능이 동작합니다.

### 방법 B: HTTPS 터널링 주소 설정 (LocalTunnel 또는 ngrok)
외부 주소(Vercel)를 꼭 유지해야 하는 경우, 로컬 서버를 임시 HTTPS 주소로 포워딩합니다. LocalTunnel은 별도의 가입이나 인증 토큰 없이 간편하게 사용할 수 있어 추천합니다.

#### Option 1: LocalTunnel 사용 (추천)
1. 터미널에서 `npx localtunnel` 명령어를 실행합니다 (Node.js 설치 필요).
   ```bash
   npx localtunnel --port 5000
   ```
   또는 특정 서브도메인을 지정하여 고정하고 싶다면:
   ```bash
   npx localtunnel --port 5000 --subdomain security-roadmap-backend
   ```
2. 발급되는 HTTPS 주소(예: `https://security-roadmap-backend.localthrough.me` 또는 임의의 `https://xxxx.localtunnel.me`)를 복사합니다.
3. 최초 접속 시 브라우저에 "Click to Continue" 경고창이 나타나면 버튼을 눌러 승인해 줍니다.

#### Option 2: ngrok 사용
1. 터미널에서 `ngrok`을 실행합니다.
   ```bash
   ngrok http 5000
   ```
2. 발급되는 HTTPS 주소(예: `https://abcd-123.ngrok-free.app`)를 복사합니다.

#### 연결 및 적용 방법
1. Vercel 사이트 상단의 **"백엔드 서버 연결 제어"** 입력창에 복사한 터널링 HTTPS 주소를 입력하고 **"연결하기"** 버튼을 누릅니다.
2. 연결 성공 후 브라우저 로컬 저장소에 주소가 기억되므로 원활하게 사용이 가능합니다.
