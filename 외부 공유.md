# 외부 공유 (Vercel 배포 및 연동) 히스토리 관리 문서

본 문서는 **Security Roadmap Agent** 시스템의 UI를 외부 사용자에게 공유하기 위해 **Vercel**을 통해 정적 사이트로 배포하고, 로컬 PC의 AI 백엔드 서버와 안전하게 통신 및 연동하기 위해 해결한 전 과정의 기술적 아키텍처와 활용 가이드를 상세히 기록합니다.

---

## 1. 배포 아키텍처 개요

### 1.1 왜 Vercel과 로컬 백엔드 조합인가?
- **외부 사용자 사용성 테스트 (UX Test)** 및 기획 시연을 신속하게 진행하기 위해 Vercel의 정적 호스팅 서비스를 프론트엔드로 채택했습니다.
- 보안 감사의 핵심 자산인 고객사 진단 체크리스트, 사전환경조사서, 자산목록 등은 매우 민감한 정보이므로 외부 클라우드에 백엔드를 통째로 올릴 수 없습니다.
- 따라서, **프론트엔드 UI만 Vercel(HTTPS)로 외부에 공개**하고, 실제 AI 매핑 및 엑셀 최종 추출을 수행하는 **REST API 백엔드(Flask, sLLM, ChromaDB)는 사용자 로컬 PC 사설망 내부에서 독립 기동**하는 하이브리드 아키텍처를 채택했습니다.

### 1.2 배포 구조 및 통신 흐름

```
┌─────────────────────────────────┐           ┌────────────────────────────────┐
│  Vercel (정적 호스팅)            │           │  로컬 PC 백엔드 (사설망)         │
│  ──────────────────             │  HTTPS    │  ───────────────               │
│  public/ (HTML/CSS/JS)          │ ────────> │  Flask API (port 5000)         │
│  - 백엔드 제어창 (URL 동적 변경)   │  터널링   │  Ollama (Gemma 2) & ChromaDB   │
│  - 1.5초 실시간 비동기 폴링       │           │  PNAMiddleware (PNA 우회)      │
└─────────────────────────────────┘           └────────────────────────────────┘
```

> [!IMPORTANT]
> Vercel 정적 사이트는 보안 암호화 프로토콜(**HTTPS**)로만 구동됩니다. 브라우저 보안 규격상 HTTPS 사이트에서 로컬망 HTTP 주소로의 직접 요청은 **Mixed Content** 정책 및 **PNA(Private Network Access)** 정책에 의해 엄격하게 차단됩니다. 이를 극복하기 위해 터널링 통신 및 사설망 우회 규격을 백엔드와 프론트엔드에 장착했습니다.

## 2. 브라우저 보안 규격 해결 핵심 이력

### 2.1 Mixed Content & HTTPS 터널링
- Vercel(HTTPS)에서 로컬 사설망 백엔드로 직접 API를 날릴 때 발생하는 보안 오류를 우회하기 위해, 로컬 백엔드를 HTTPS 터널링 도구(`localtunnel` 또는 `ngrok`)로 연동하여 공인 HTTPS 도메인을 일시 부여하는 아키텍처를 수립했습니다.
- LocalTunnel 기동 명령어:
  ```bash
  npx localtunnel --port 5000
  ```
- 터널링 주소가 발급되면, 사용자는 Vercel 프론트엔드 화면 상단의 연결 제어판에 터널링 HTTPS 도메인(예: `https://chilly-jokes-trade.loca.lt`)을 기입하여 연동합니다.

### 2.2 Chrome PNA (Private Network Access) 사설망 차단 해결
- 크롬 브라우저가 외부 도메인(Vercel)에서 사설 IP망(Localhost 또는 터널링된 로컬망)에 접근하는 것을 탐지하여 보안 경고를 뿜어내는 PNA 이슈가 발생했습니다.
- 이를 해결하기 위해 백엔드 응답에서 사설망 접근 승인 요청 예비 필터(`Preflight request`)에 대한 대응으로 `Access-Control-Allow-Private-Network: true` 헤더를 내려주어야 합니다.
- **트러블슈팅**: Flask의 `@app.after_request`나 일반 CORS 라이브러리를 사용 시 해당 헤더 값이 중복으로 들어가 크롬 브라우저가 승인을 거부하는 현상이 발생했습니다.
- **해결 방안**: Flask 애플리케이션의 WSGI 최상위 레이어에 **`PNAMiddleware` 미들웨어 클래스를 직접 탑재**하여, 기존 헤더 중복을 제거하고 명확하게 단일 문자열 `"true"`로 고정 전송하도록 조치함으로써 크롬 브라우저의 PNA 차단을 완벽히 우회하는 데 성공했습니다.

### 2.3 LocalTunnel 60초 포트 타임아웃 극복 (비동기화 및 폴링 도입)
- LocalTunnel 중계 게이트웨이는 요청 대기 시간이 60초를 넘어가면 무조건 **504 Gateway Timeout** 에러를 브라우저로 반환합니다.
- AI 기반 로드맵 분석 매핑은 결함 개수에 따라 최소 1분에서 최대 3분 이상 소요되므로, 기존 동기식 POST 요청은 100% 타임아웃 오류를 유발했습니다.
- **해결 방안**:
  - `/api/map` API를 **비동기식 스레드 백그라운드 기동 방식**으로 전면 재설계했습니다. 클라이언트가 분석 요청 시, 작업 고유 번호(`task_id`)를 즉시 반환하고 연결을 바로 끊습니다.
  - 백엔드에 현재 비동기 작업의 진행도와 매핑 완료 건수를 보관하는 상태 확인용 엔드포인트 `/api/map/status/<task_id>`를 신설했습니다.
  - 프론트엔드 자바스크립트는 더미 타이머를 걷어내고, **1.5초 주기로 백엔드의 작업 상태를 실시간 폴링(Polling)**하여 진행률 바(progress bar)와 완료 건수를 연동해 렌더링하도록 수정함으로써 타임아웃 오류를 원천 타개했습니다.

## 3. 로컬 백엔드 연동 및 외부 테스트 가이드

외부 테스트 사용자나 팀원들이 Vercel 사이트를 통해 로컬 AI 백엔드에 안전하게 연결하기 위한 단계별 행동 지침입니다.

### Step 1: 로컬 백엔드 및 LocalTunnel 기동
1. 로컬 개발 환경에서 백엔드 서버를 켭니다:
   ```bash
   python app.py
   ```
2. 새 터미널을 열고 LocalTunnel을 기동해 외부 공인 HTTPS 주소를 부여받습니다:
   ```bash
   npx localtunnel --port 5000
   ```
   (출력 예시: `your url is: https://chilly-jokes-trade.loca.lt`)

### Step 2: 터널링 최초 접속 본인인증 승인 (중요)
- LocalTunnel 보안 규정상, 발급된 HTTPS 터널 주소에 최초 1회 수동 접속하여 보안 승인을 완료해야 프론트엔드가 백엔드 API 데이터를 수신할 수 있습니다.
1. 브라우저에서 발급된 LocalTunnel 주소(예: `https://chilly-jokes-trade.loca.lt`)로 직접 접속합니다.
2. 화면에 나타나는 본인인증 확인 페이지에서 **사용자의 현재 공인 IP 주소**를 입력하라는 창이 나타납니다.
   - 사용자 본인의 공인 IP(예: `210.182.247.5`)를 정확히 입력합니다.
3. 아래의 **[Click to Continue]** 또는 **[Submit]** 버튼을 클릭하여 로컬 서버의 JSON 환영 인사가 출력될 때까지 확인합니다.

### Step 3: Vercel 정적 사이트에서 연결
1. Vercel 배포 주소인 [https://security-loadmap-ax.vercel.app](https://security-loadmap-ax.vercel.app) 에 접속합니다.
2. 화면 우측 상단의 **[백엔드 서버 제어 센터]** 패널 입력 필드에 Step 1에서 발급받은 LocalTunnel HTTPS 도메인 주소(예: `https://chilly-jokes-trade.loca.lt`)를 붙여넣습니다.
3. **[연결하기]** 버튼을 클릭합니다.
4. `"서버 연결 성공!"` 알림 배지가 활성화되면 외부 연동 준비가 완료된 것입니다. 상세체크리스트, 사전환경조사서, 자산목록을 마음껏 업로드하여 비동기 AI 분석 및 고도화된 엑셀 가시성 최종 추출 기능을 테스트할 수 있습니다.
