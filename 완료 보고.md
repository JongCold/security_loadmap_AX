# [완료 보고] 로컬 LLM 기반 보안 솔루션 로드맵 자동 수립 솔루션

본 문서는 로컬 LLM(Gemma 2) 및 웹 인터페이스를 탑재한 **"보안 솔루션 로드맵 자동 수립 에이전트"** 솔루션의 개발 완료 내역 및 상세 사용 설명서입니다.

---

## 1. 솔루션 구성 및 아키텍처

본 솔루션은 비연결성 내부 폐쇄망 환경에서도 정보보안 컨설팅 업무의 자동화(AX)를 실현할 수 있도록, **로컬 LLM 서비스(Ollama)**와 **웹 UI 인터페이스(Flask)**를 결합하여 독립 작동이 가능하도록 설계 및 구현되었습니다.

### 1.1 솔루션 디렉터리 구조
개발된 솔루션의 전체 디렉터리 레이아웃 구조는 다음과 같습니다.

```text
c:\Security loadmap_Auto\
├── app.py                      # Flask 웹 서버 및 백엔드 API (엔트리포인트)
├── roadmap_agent_llm.py        # 로컬 Gemma 2 LLM 연동, 빨간색 결함 감지 및 엑셀 Cloner 코어
├── templates\
│   └── index.html              # 프론트엔드 UI 웹 페이지 (HTML 마크업)
├── static\
│   ├── css\
│   │   └── style.css           # Vanilla CSS (다크모드 및 글래스모피즘 프리미엄 디자인)
│   └── js\
│       └── app.js              # Vanilla JS (비동기 API 처리 및 UI 렌더링)
├── .env.template               # 환경설정 템플릿 파일
└── 완료 보고.md                # [본 파일] 최종 개발 완료 보고서
```

## 2. 솔루션 핵심 기능 명세

1. **빨간색 글씨(결함 사항) 탐지 엔진**:
   - 업로드된 상세체크리스트의 `내부보안감사체크리스트` 시트를 스캔합니다.
   - 단순 결함 평가(X) 뿐만 아니라, `운영현황 및 증적` 및 `개선방안` 열에서 컨설턴트가 **조치가 필요하여 빨간색 글씨(`FFFF0000`)**로 기술한 텍스트가 있는 셀들을 폰트 ARGB 메타데이터 검출을 통해 정밀 선별하여 매핑 대상으로 삼습니다.
2. **로컬 LLM (Gemma 2 - gemma4:e4b) RAG 엔진**:
   - 로컬에 구축된 Ollama의 `gemma4:e4b` 모델과 REST API로 통신합니다.
   - 자사 보안 솔루션 DB(152개 명세 데이터)를 RAG 컨텍스트로 제공하여, 결함 내용에 최적화된 솔루션을 지능적으로 1:1 매치합니다.
   - Ollama의 `json` 모드를 강제하여 누락 없는 정형 구조화 데이터(Structured Output)를 획득합니다.
3. **서식 보존형 엑셀 쓰기 (Style Cloner)**:
   - 엑셀 파일을 새로 쓸 때 기존의 선 두께, 폰트 굵기, 정렬, 셀 배경색이 파괴되는 문제를 방지하기 위해 `openpyxl` 스타일 속성을 타겟 셀로 그대로 복사(Cloning)한 후 텍스트 데이터만 안전하게 덮어씁니다.
4. **프리미엄 반응형 웹 UI**:
   - 딥 네이비와 네온 사이언 블루의 조화, 그리고 투명 유리 질감을 나타내는 글래스모피즘(Glassmorphism) 스타일의 Vanilla CSS를 구현했습니다.

## 3. 솔루션 상세 사용 방법

### 3.1 사전 기동 조건
1. **Ollama 서비스 기동 및 모델 확인**:
   로컬 PC에 Ollama가 가동 중인지 확인하고, 설치된 모델명에 `gemma4:e4b`가 있는지 점검합니다.
   ```bash
   ollama list
   ```
2. **Flask 백엔드 서버 실행**:
   프로젝트 루트 폴더(`c:\Security loadmap_Auto`)에서 아래 명령어를 실행하여 웹 서버를 기동합니다.
   ```bash
   python app.py
   ```
3. **브라우저 접속**:
   서버가 기동되면 웹 브라우저를 열고 `http://localhost:5000` 주소로 접속합니다.

### 3.2 웹 UI 조작 가이드

```
┌──────────────────────────────────────────────────────────┐
│              SECURITY ROADMAP AGENT (AX)                 │
├──────────────────────────────────────────────────────────┤
│  [1. 체크리스트 업로드]                                  │
│  ┌────────────────────────────────────────────────────┐  │
│  │    클릭하여 엑셀 업로드 또는 파일 드래그 앤 드롭   │  │
│  └────────────────────────────────────────────────────┘  │
│    └─ (업로드 성공: 8건의 결함이 감지되었습니다.)        │
│                                                          │
│  [ ▶ AI 매핑 및 스케줄링 기동 ]                          │
│                                                          │
│  [2. 매핑 결과 종합 보고] -> 매핑 진행 시 테이블 활성화 │
│  ┌────────────────────────────────────────────────────┐  │
│  │ No | 항목명 | 결함사항 | 과제명 | 예산 | 로드맵연도 │  │
│  │ 1  | 1.정책 | 정책미반 | ISMS   | 1.5천| 2026년     │  │
│  │ ...                                                │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  [ ⬇ 최종 엑셀 로드맵 추출 ]                             │
└──────────────────────────────────────────────────────────┘
```

1. **상세체크리스트 업로드**:
   - 파일 업로드 영역을 클릭하거나 엑셀 파일(`2026년 OO기기업명_내부보안점검_상세체크리스트_테스트 파일.xlsx`)을 드래그앤드롭합니다.
   - 업로드 즉시 빨간색 글씨 셀 파싱 모듈이 작동해 결함 개수를 카운트하여 화면에 표시합니다.
2. **매핑 시작 버튼**:
   - **`[AI 매핑 및 스케줄링 기동]`** 버튼을 클릭합니다.
   - 로딩 스피너 및 프로그레스 바가 활성화되며, 로컬 Gemma 2 LLM이 RAG 기반 실시간 추론과 스케줄링 연산(시급성/위험도 스코어 계산)을 비동기로 처리합니다.
3. **매핑 결과 종합 보고**:
   - 연산이 완료되면 UI에 로드맵 테이블이 활성화되어, 매칭된 솔루션명, 예상 예산, 법적 준거성 여부, 배정된 연도 배지가 시각적으로 표시됩니다.
4. **최종 엑셀 추출**:
   - 우측 상단의 **`[최종 엑셀 로드맵 추출]`** 버튼을 클릭합니다.
   - 서버가 `{오늘날짜}_{기업명}_정보보안컨설팅_보안솔루션 로드맵.xlsx` (예: `20260528_OO기기업명_정보보안컨설팅_보안솔루션 로드맵.xlsx`) 파일명의 서식 보존 결과 파일을 빌드하여 자동으로 다운로드 창을 띄웁니다.
