# 4차 개발 완료 보고서 (저장소 및 RAG 데이터 관리 제어 센터 개발)

본 보고서는 **정보보안감사 보안솔루션 로드맵 자동 수립 시스템**의 4차 개발 단계에서 수행된 디스크 저장소 최적화, ChromaDB RAG 데이터베이스 제어 기능 추가 및 타입 안정성 강화 조치의 최종 결과를 기술합니다.

---

## 1. 개발 개요
* **목표**: 
  - 서버 측 리소스(저장소 및 메모리)의 효율적인 관리와 정보 유출 방지를 위한 **업로드 파일 일괄 정리** 기능 구현.
  - 자사 보안 솔루션의 추가, 수정, 삭제 등의 변동 요인에 유연하게 대처할 수 있도록 **RAG 벡터 데이터베이스 지식 베이스 초기화(삭제)** 기능 개발.
  - **서버 리소스 및 지식 DB의 적재 현황 실시간 시각화(모니터링)** 기능 구현.
  - 다양한 데이터셋 입력에 대한 브라우저 런타임 예외를 차단하는 **프론트엔드 타입 안정성(Type-Safety) 가드** 강화.
* **수행 기간**: 2026년 6월 1일
* **핵심 구현체**:
  - `rag_engine.py` (`clear_index()` 인덱스 삭제 메소드 탑재)
  - `app.py` (임시 파일 삭제 `/api/uploads/clear`, RAG 삭제 `/api/rag/clear`, 목록 조회 `/api/uploads/list`, `/api/rag/list` 백엔드 엔드포인트 신설/확장)
  - `templates/index.html` (데이터 관리 제어 센터 내 현황 보기 토글 및 스크롤 테이블 UI 카드 추가)
  - `static/js/app.js` (제어 센터 비동기 API 연동, 현황 조회 동적 렌더링 및 타입 가드 보완)

## 2. 4차 개발 핵심 항목 및 구현 내용

### ① 업로드 임시 파일 일괄 정리 ( uploads 폴더 비우기 )
* **요구 분석**: 사용자가 업로드하는 진단 체크리스트 엑셀 파일은 기업의 민감한 결함 내역을 포함하고 있어, 분석 완료 후 보관 필요성이 떨어지면 즉각 삭제할 수 있어야 합니다.
* **백엔드 구현**: `/api/uploads/clear` API를 신설하여, 업로드 폴더(`uploads`) 내부의 모든 임시 엑셀 파일들을 서버 디스크 상에서 안전하게 영구 삭제하도록 처리했습니다.
* **프론트엔드 연동**: 클릭 시 `confirm` 컨펌 창을 통해 의도치 않은 삭제를 예방하고, 삭제 완료 시 프론트엔드의 업로드 상태(`uploadedData`) 및 결과 테이블 표를 초기화(Clear)하도록 유기적으로 결합했습니다.

### ② RAG 지식 베이스 초기화 ( ChromaDB 데이터셋 삭제 )
* **요구 분석**: 자사 보안 솔루션 리스트 엑셀 데이터에 변동이 생기거나, 테스트용 임베딩을 비우고 신규 솔루션 리스트만 적재하고 싶을 때, ChromaDB에 기구축된 벡터 청크들을 리셋하는 물리적 초기화 수단이 요구되었습니다.
* **백엔드 구현**: RAG 엔진 클래스(`SecuritySolutionRAG`) 하단에 `clear_index()` 메소드를 설계하여 ChromaDB 내의 해당 컬렉션 데이터를 완전 삭제하도록 연동하고, 이를 호출하는 `/api/rag/clear` API를 개설했습니다.
* **프론트엔드 연동**: 클릭 시 강력한 유실 경고 모달(`confirm`)을 띄워 관리자가 실수를 방지하도록 가이드하고 비동기 요청을 전달하게 했습니다.

### ③ 데이터 렌더링 타입 가드 강화 (서버 통신 실패 완벽 방지)
* **이슈 해결**: 이전 체크리스트 엑셀 파싱 및 화면 렌더링 중, 로드맵연도가 숫자로 기입되어 있거나 공백인 상태인 경우 자바스크립트의 `.includes()` 연산에서 `TypeError`를 던져 통신 에러를 유발하는 현상을 해결했습니다.
* **가드 적용**: `app.js` 렌더러 함수에 `String(res.로드맵연도 || '')` 형변환 가드를 적용하고, `parseInt(res.시급성, 10)` 가드를 추가 적용하여 어떠한 유동적인 형태의 데이터가 인풋으로 오더라도 자바스크립트 예외가 차단되도록 처리했습니다.

### ④ 서버 저장소 및 RAG 데이터 실시간 모니터링 조회 (신규 추가)
* **요구 분석**: 현재 서버 디스크에 어떤 민감한 임시 파일이 보관되어 있는지 실시간으로 모니터링하고, RAG 벡터 DB에 실제로 어떤 보안 솔루션들이 적재되어 매핑 대상으로 활용 중인지 조회할 수 있는 투명한 인터페이스가 요구되었습니다.
* **백엔드 구현**: 
  - `/api/uploads/list`: `uploads` 폴더 내 저장된 파일의 파일명, 파일 크기, 업로드 일시를 실시간으로 탐색하여 JSON 배열로 반환합니다.
  - `/api/rag/list`: ChromaDB 컬렉션에 임베딩된 전체 문서 수와 각각의 메타데이터(ID, 보안영역, 솔루션구분, 제조사, 제품명 및 기능 설명 요약) 목록을 안전하게 파싱하여 반환합니다.
* **프론트엔드 연동**:
  - '현황 보기' / '현황 닫기'의 직관적인 토글 컨트롤을 제공합니다.
  - 데이터를 펼쳤을 때 비동기 통신으로 최신 목록을 가져와 프리미엄 다크테마 스타일의 스크롤 테이블 영역에 깨끗하게 렌더링합니다.
  - 파일 삭제나 RAG 초기화 작동 시, 목록 패널이 열려 있다면 즉시 변경 사항을 실시간 반영(비워짐 표시)하도록 상호 연동했습니다.

### ⑤ 엑셀 다운로드 보안 경고 원천 해결 (Form POST 스트림 다운로드 방식 도입)
* **이슈 사항**: 기존 JavaScript의 Fetch API 기반 Blob URL 생성 다운로드 방식은 크롬 등 브라우저의 '세이프 브라우징' 보안 검열에 걸려 "안전하지 않은 다운로드 차단" 경고를 발생시켰습니다.
* **해결 조치**: 
  - 프론트엔드에서 Fetch Blob URL 생성을 중단하고, 동적 HTML `<form>`을 생성하여 `/api/export`로 직접 데이터를 Submit하는 네이티브 Form POST 스트림 다운로드 방식으로 전환했습니다.
  - 백엔드(`/api/export`) API의 반환 헤더에 `X-Content-Type-Options: nosniff`, `Cache-Control` 등 보안 규격을 엄격하게 주입하여 브라우저 수준에서 신뢰할 수 있는 보안 파일 다운로드 스트림으로 처리하여 경고 문구를 원천 차단했습니다.

### ⑥ 적용 완료 상태 고정 및 실시간 리셋 트리거 연동
* **개선 사항**: 기존에는 "입력 완료" 클릭 시 1.5초 후 다시 "입력 완료" 상태로 돌아가 현재 해당 행의 저장 여부를 직관적으로 알기 어려웠습니다.
* **적용 내역**:
  - `setTimeout` 타이머를 제거하여 "적용됨" 버튼 클릭 시 상태가 `[적용됨] (녹색 테마)`으로 영구 고정되게 개선했습니다.
  - 사용자 편의를 위해, 해당 행의 `위험도`, `예상 예산`, `로드맵 연도` 중 어느 하나라도 값이 재차 변경되면 자동으로 상태 버튼이 `[입력 완료] (블루 아웃라인)`로 실시간 롤백되는 스마트 리셋 트리거를 연동하여 데이터 변경과 저장 상태의 정합성을 보장합니다.

### ⑦ 최종 엑셀 추출 시 세부 체크리스트 열 병합 자동화 복원
* **개선 사항**: RAG 분석 데이터의 입력 정밀도를 위해 파싱 단계에서는 병합을 풀었으나, 이를 다운로드 가능한 최종 결과물로 추출할 때는 인쇄 및 가독성을 위해 각 열들이 보기 좋게 가로 병합되어 있어야 합니다.
* **적용 내역**:
  - `roadmap_agent_llm_new.py` 내의 `generate_roadmap_excel` 저장부 루프에 자동 열 병합 로직을 추가했습니다.
  - 데이터가 작성될 때마다 **세부점검내용 (Column C-D)**, **운영현황 및 증적 (Column E-F)**, **개선방안 (Column G-H)** 영역이 가로 방향으로 완벽하게 병합 처리되며, 원본 템플릿의 셀 테두리 및 폰트 서식은 100% 보존되도록 구현했습니다.

### ⑧ 임시 파일 실명 및 업로드 일시 보존 (식별 편의성 고도화)
* **이슈 사항**: 기존에는 업로드 시 UUID 암호화 파일명으로 디스크에 저장되어 제어 센터 내에서 파일 구분 및 지워도 되는 파일인지 여부를 구별하기 매우 곤란했습니다.
* **해결 조치**: 
  - `app.py`에 한글 유실 없는 `make_safe_filename()` 파일명 정제 및 일시 각인 헬퍼 함수를 구축했습니다.
  - 저장 시 **"업로드일시(YYYYMMDD_HHMMSS)_원본파일명"** (예: `20260601_092622_2026년_OO제로인_내부보안점검_상세체크리스트_테스트_파일.xlsx`) 형태로 안전하게 실명을 살려 보존합니다.
  - 이를 통해 관리 제어 센터의 목록에서 파일의 성격을 즉각 직관적으로 식별 및 통제할 수 있습니다.

### ⑨ 결함 판정 조건의 정형성 고도화 (공식 평가 'X' 기준 100% 매칭 수립)
* **개선 사항**: 기존의 빨간 글씨(Red-Font) 색상이나 권고/권장 등의 텍스트 어조(Semantic) 분석 기반 판정은 체크리스트 작성자의 서식 누락이나 스타일 해제 등 유동적인 요인으로 인해 결함 선별에 오류를 발생시키는 원인이 되었습니다.
* **해결 조치**:
  - `roadmap_agent_llm_new.py`의 결함 탐색 파이프라인(`parse_red_cells_from_checklist`)을 개편했습니다.
  - 임의의 색상 스타일 판정을 배제하고, 상세체크리스트 4번 시트인 `내부보안감사체크리스트` E열(평가)이 정확하게 조치 대상인 **"X"**로 기록된 모든 행을 1차 결함으로 추출하도록 판정 알고리즘을 단일화했습니다.
  - 이를 통해 폰트 색상이나 비공식 서식 스타일 변경으로 인한 결함 누락 오류를 완전히 원천 배제하였으며, 감사의 공식 결과를 RAG 지식 매핑으로 이어주는 정확성 100%의 분석 데이터를 수집합니다.

### ⑩ 병합 셀 구조 하에서의 결함 파싱 누락 방지 (항목명 상속 알고리즘 적용)
* **이슈 사항**: 4번 시트 `내부보안감사체크리스트`에서 C열(항목명)이 여러 개의 세부점검내용 행에 걸쳐 세로로 병합되어 있는 경우, `openpyxl` 라이브러리는 첫 행을 제외한 나머지 하위 셀들을 `None`(빈값)으로 반환했습니다. 이에 따라 항목명이 비어있어 1차 결함(평가 X)으로 수집되지 못하고 스킵되는 현상이 있었습니다. (예: `OO파이낸셜` 상세체크리스트의 실결함 7개 중 5개가 누락되는 문제 식별)
* **해결 조치**:
  - 파싱 루프가 돌아갈 때 직전의 유효한 항목명을 메모리에 임시 기억하는 `last_item_name` 변수를 설계했습니다.
  - 루프 진행 중 C열의 셀 값이 `None`이거나 공백일 경우, 직전 행에서 읽은 항목명을 그대로 물려받아 바인딩하도록 **항목명 상속(Forward Fill)** 파싱 방식을 적용했습니다.
  - 이를 통해 엑셀 시트 내의 임의의 대분류 병합 패턴에도 결함 탐색 누락률 0%를 기록하며, 데이터 파싱의 견고성(Robustness)을 대폭 향상했습니다.

## 3. UI/UX 디자인 요소 및 프리미엄 스타일링
* **Glassmorphism 카드 적용**: 기존 세련된 다크 모드 테마에 맞춰 투명도 배경(`rgba(255,255,255,0.03)`)과 부드러운 블러 처리가 들어간 저장소 제어 센터 카드를 배치했습니다.
* **실시간 데이터 현황 패널**: 모니터링 테이블이 보여지는 영역은 내부 스크롤(`max-height: 250px; overflow-y: auto;`)을 적용하고, Sticky 헤더 스타일링을 주어 데이터가 많더라도 사용자 조작의 사용성을 대폭 끌어올렸습니다.
* **그라데이션 액션 버튼**: 
  - 임시 파일 정리 버튼: 경고와 활력을 주는 오렌지-옐로우 그라데이션(`linear-gradient(135deg, #f2994a, #f2c94c)`) 적용.
  - RAG 초기화 버튼: 위험과 경고를 뜻하는 레드 그라데이션 및 아웃라인 붉은 발광 효과(`linear-gradient(135deg, #eb5757, rgba(0,0,0,0))`)를 적용해 제어의 가시성을 한껏 프리미엄하게 끌어올렸습니다.

## 4. 최종 통합 검증 결과
* **업로드 파일 정리 & 실시간 목록**: 임시 파일 일괄 비우기 버튼 클릭 시 uploads 디렉토리가 클리어됨과 동시에, 현황 보기 테이블에서도 "보관된 임시 파일이 없습니다."로 실시간 갱신됨을 성공적으로 확인했습니다.
* **RAG 초기화 & 현황 조회**: 초기화 버튼 가동 시 ChromaDB 내의 컬렉션 카운트가 `0`으로 비워지고, 현황 조회 패널 내에서도 "임베딩된 RAG 지식 데이터가 없습니다."로 즉각 리셋됨을 실증했습니다.
* **통신 장애**: 새로운 엑셀 파일들을 대상으로 연속적인 업로드 및 매핑을 테스트하여, 통신 에러 없이 실시간으로 테이블과 데이터가 상호 로드되는 것을 확인했습니다.

## 5. [고도화] 병합 셀 무결성 및 비고란 누락 방지 조치 (추가 완료)
* **병합 파손 원인 해결 (통합 오프셋 병합 알고리즘)**:
  - 정보보호시스템 자산 개수($N$)와 로드맵 매핑 결과 수($M$)에 따라 행의 추가/삭제가 동적으로 일어나며 openpyxl의 행 조정(`insert_rows`/`delete_rows`) 시 이미 지정된 병합 범위가 꼬이거나 풀려 레이아웃이 깨지던 현상을 완벽히 해결했습니다.
  - 엑셀 조작 기동 시 원본 시트 내의 모든 병합 셀 정보를 메모리에 백업하고 **전체 영역을 언머지(unmerge)**한 상태에서 데이터를 기입 및 행 조정을 완료한 뒤, 백업된 범위를 오프셋(`diff` 및 `roadmap_diff`)에 따라 통합 연산하여 **최종적으로 단 1회 일괄 복구 적용**하는 견고한 알고리즘을 수립했습니다.
* **로드맵 매핑 영역 및 비고란 병합 수정**:
  - 로드맵 데이터 각 행의 세부점검내용(C-D), 운영현황 및 증적(E-F), 개선방안(G-H)에 대해 가로 병합을 누락 없이 적용했습니다.
  - 자산목록 데이터 행의 비고란(N-O-P) 병합은 확실히 적용하되, 로드맵 매핑 영역의 14~16열(예상예산~비고)이 과도하게 병합되어 로드맵연도나 비고 정보가 소실되던 오류를 수정하여 각 열의 데이터가 온전히 출력되도록 정정했습니다.
* **비고란 데이터 폴백(Fallback) 안전장치 수립**:
  - RAG 및 LLM 매핑 처리 도중 API 예외나 데이터 누락으로 인해 비고(`note_val`)가 빈 값이나 `None`으로 기입되는 현상을 방지하는 자동 보정 코드를 이식했습니다.
  - 매핑 결과 추천 솔루션의 유무에 따라 감사의 신뢰성을 높여주는 표준 권고 포맷(`[추천 솔루션] {제조사} - {제품명}\n[선정 이유] {이유}`)의 문구를 자동 완성하여 채워넣음으로써 분석 보고서의 완성도를 항시 일관되게 보증합니다.
