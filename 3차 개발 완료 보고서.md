# 3차 개발 완료 보고서 (RAG 시스템 구축 및 매핑 정밀도 고도화)

본 보고서는 **정보보안감사 보안솔루션 로드맵 자동 수립 시스템**의 3차 개발 단계에서 수행된 ChromaDB 기반 RAG(검색 증강 생성) 시스템 구축 및 매핑 알고리즘 고도화 작업의 최종 결과를 상세히 기술합니다.

---

## 1. 개발 개요
* **목표**: 이전 시스템의 무분별한 외산 제품 추천(할루시네이션) 문제를 원천 차단하고, 자사 보안 솔루션 풀(100% 매칭 기준)에 기반하여 진단 결함 사항별 최적의 자사 솔루션을 1:1로 정밀 매핑하는 RAG 파이프라인 개발
* **수행 기간**: 2026년 5월 29일
* **핵심 구현체**: 
  - `rag_engine.py` (ChromaDB 기반 RAG 검색 및 전처리 엔진 신규 구축)
  - `roadmap_agent_llm_new.py` (RAG 연동 및 프롬프트 하이브리드 오케스트레이션 개정)
  - `app.py` (서버 구동 시 RAG 자동 임베딩 및 인덱스 재구축 API 탑재)

---

## 2. 3차 개발 핵심 항목 및 구현 내용

### ① ChromaDB 기반 RAG 벡터 데이터베이스 구축
* **Markdown 세그먼트 가공**: 자사 보안 솔루션 엑셀 파일 내의 로우(Row) 데이터를 LLM이 문맥적으로 파악하기 용이하도록 타이틀, 보안영역, 솔루션 구분, 제조사명, 제품명, 기능설명으로 구조화된 마크다운 형식으로 가공하여 저장
* **ChromaDB 임베딩**: ChromaDB의 내장 임베딩 모델을 사용하여 텍스트 데이터의 시맨틱 벡터 변환 및 로컬 영구 디렉토리(`chroma_db`)에 캐싱 처리 완료

### ② 한글 매핑 정밀도 고도화를 위한 하이브리드 Re-ranking 알고리즘 개발
* **임베딩 모델의 한계 극복**: 기본 임베딩 모델(all-MiniLM-L6-v2)의 영어 편향적 한계로 인한 한국어 시맨틱 검색 왜곡 현상 발견
* **하이브리드 점수 모델**: 벡터 기반 코사인 유사도(30% 가중치)와 형태소/명사 기반 단어 포함 가중치(70% 가중치)를 합산하여 최종 유사도 스코어를 계산하는 Re-ranking 로직 도입
* **핵심 메타 가산점**: 결함 쿼리 내의 핵심 보안 단어가 제품의 `솔루션 구분`, `제품명`, `보안영역`에 포함되어 있는 경우 최대 `+0.5`점의 가산점 보너스를 부여하여 방화벽, 백신, 개인정보필터링 등 핵심 도메인이 한치의 오차도 없이 상위 매칭되도록 설계

### ③ RAG 데이터 노이즈 차단 및 클렌징 필터링
* **비제품군 서비스 제거**: 자사 리스트 내부의 컨설팅 법인(KPMG, 삼정, 딜로이트 등)이나 수동 교육, 단순 유지보수, 보안 관제 서비스 등 장비 도입 대상이 아닌 비제품군 데이터(약 50건)를 사전 필터링 규칙을 통해 RAG 임베딩 과정에서 완전히 격리 (총 149개 행 중 순수 도입 가능 솔루션 99건으로 정합성 최적화)
* **제외 키워드 검사**: 제품명이나 설명에 "제외함", "보류함", "가능성 낮음", "보류" 등이 명시된 유보 대상 솔루션 자동 스킵

### ④ 백엔드 RAG 제어 API 설계 및 통합
* **RAG Rebuild 엔드포인트 (`POST /api/rag/rebuild`)**: 향후 관리자가 자사 솔루션 리스트 엑셀 파일을 수정한 뒤, Flask 서버 재구축 없이 웹 상에서 실시간으로 RAG 벡터 데이터베이스를 다시 빌드할 수 있는 강제 재구축 API 탑재
* **RAG Status 엔드포인트 (`GET /api/rag/status`)**: 현재 벡터 DB에 임베딩되어 있는 실시간 자사 솔루션 개수 및 RAG 엔진의 정상 로드 여부를 모니터링할 수 있는 상태 확인 API 추가

## 3. 최종 매핑 품질 검증 결과
구현된 하이브리드 RAG 엔진을 대상으로 주요 대표 진단 쿼리를 투입하여 정밀도를 실증 검증한 결과는 다음과 같습니다.

* **테스트 1: '방화벽 네트워크 보안 침입 차단'**
  - **매칭 1위**: `Fortinet - FG 시리즈` (유사도 81.32%) - 차세대 방화벽 제품 정확하게 랭크인
  - **매칭 2위**: `펜타시큐리티 - Wapples` (유사도 80.97%) - 웹 방화벽 제품이 정확히 추종 매칭

* **테스트 2: '개인정보 접속기록 관리 로그 감사'**
  - **매칭 1위**: `피앤피시큐어 - INFOSAFER` (유사도 86.70%) - 개인정보 접속기록 통제 솔루션 정확하게 매칭
  - **매칭 2위**: `위즈코리아 - WEEDS BlackBox Suite` (유사도 81.83%) - 업계 1위 접속기록 관리 솔루션이 누락 없이 추종 매칭

* **테스트 3: '악성코드 안티바이러스 엔드포인트 보안'**
  - **매칭 1위**: `시큐브 - 악성코드 통제, 랜섬웨어` (유사도 87.31%) - 엔드포인트 악성코드 차단 장비 완벽 매칭

> [!NOTE]
> RAG 시스템 구축 이후, 자사 솔루션 리스트에 없는 임의의 외산 제품(예: Cisco, Snyk 등)이나 이상한 플레이스홀더 텍스트(예: `[실제 제품명]`, `[사용자 정의]`)가 최종 보고서 및 엑셀에 혼입되는 할루시네이션 현상이 **0%로 완전히 해소**되었습니다.

## 4. 향후 시스템 유지관리 방안
* **자사 솔루션 업데이트 시**:
  - `자사 보안 솔루션 리스트 (1).xlsx` 파일을 변경한 후, 브라우저 또는 API 호출 툴을 통해 `http://localhost:5000/api/rag/rebuild` 주소로 `POST` 요청을 보내는 것만으로 즉시 실시간 임베딩 데이터 갱신 가능
* **모델 교체 및 이관**:
  - Ollama 로컬 모델 사양 변경 시 `app.py` 및 `roadmap_agent_llm_new.py`의 `OLLAMA_MODEL` 변수값 수정을 통해 즉각적인 유연한 이관 가능
